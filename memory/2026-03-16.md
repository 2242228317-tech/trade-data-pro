### Clawith 服务修复
- **问题**: Clawith 网页打不开，后端服务启动失败
- **原因**: 
  1. PostgreSQL 特有的 SQL 语句 `ALTER TYPE channel_type_enum ADD VALUE` 在 SQLite 上不兼容
  2. Windows GBK 编码不支持 emoji 字符（✅ ⚠️ ⛔）
- **修复**:
  1. 修改 `backend/app/main.py` 第 94-102 行，只在 PostgreSQL 数据库时执行 ALTER TYPE
  2. 替换所有 emoji 为 ASCII 字符：[OK] [WARN] [ERR]
- **服务状态**: 
  - 后端: http://localhost:8008 ✅
  - 前端: http://localhost:3008 ✅

### 修复的代码
```python
# 修改前：直接执行 PostgreSQL 特有的语句
await conn.execute(
    __import__("sqlalchemy").text(
        "ALTER TYPE channel_type_enum ADD VALUE IF NOT EXISTS 'atlassian'"
    )
)

# 修改后：只在 PostgreSQL 时执行
try:
    db_url = str(engine.url)
    if "postgresql" in db_url:
        await conn.execute(
            __import__("sqlalchemy").text(
                "ALTER TYPE channel_type_enum ADD VALUE IF NOT EXISTS 'atlassian'"
            )
        )
except Exception:
    pass  # Ignore errors for non-PostgreSQL databases
```
